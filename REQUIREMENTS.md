# 问剑长生 - 神通升级规划工具

## 项目概述

为手游《问剑长生》开发一个跨平台桌面应用（macOS / Windows），帮助玩家规划神通升级路径。用户输入当前所有神通的状态和目标，应用输出按周的最优操作步骤。

技术栈：**Tauri 2**

---

## 游戏数据模型

### 境界（6 个）

| 序号 | 境界名 |
|------|--------|
| 1    | 人界一 |
| 2    | 人界二 |
| 3    | 返虚   |
| 4    | 合体   |
| 5    | 大乘   |
| 6    | 渡劫   |

### 职业

- **通用职业**（所有 6 个境界均有）：剑、火、雷
- **百族职业**（仅返虚、合体、大乘 3 个境界有）

### 获取商店（4 + 1）

每个通用职业的 4 个神通分别来自不同商店：

| 商店   | 适用职业       |
|--------|---------------|
| 论剑   | 剑、火、雷     |
| 诸天   | 剑、火、雷     |
| 宗门   | 剑、火、雷     |
| 道蕴   | 剑、火、雷     |
| 百族   | 百族（专属）   |

### 神通总数

- 通用：6 境界 × 3 职业 × 4 神通 = **72**
- 百族：3 境界 × 1 职业 × 4 神通 = **12**
- **合计：84 个神通**

### 神通命名规则

`{境界}·{职业}·{商店}`，例如：`返虚·剑·论剑`、`合体·百族·百族`

## 转换机制

- **作用**：将一个神通的本体书页转换为同商店另一个神通的本体书页
- **限制**：仅限**同商店**内转换（如 `返虚·剑·论剑` ↔ `大乘·雷·论剑`，但不能转为 `大乘·雷·诸天`）
- **比例**：1:1 无损（投入 40 张，获得 40 张）
- **数量**：每次固定转换 **40 张**
- **频率**：每周免费转换次数可自定义（默认 **3 次**），跨所有商店共享次数
- **转换石**：消耗 1 个转换石可额外执行 1 次转换，不受每周免费次数限制，使用次数无上限

## 收入模型

### 战斗神通收入

每个战斗神通拥有独立的收入周期，按周期获得本体书页。收入由两个参数控制：

- **收入周期**（`incomeCycleWeeks`）：每 N 周获得一次收入
- **收入批次**（`incomeBatchCount`）：每次获得 M × 40 张本体书页

不同商店的默认收入周期：

| 商店   | 默认周期  | 默认批次 | 说明               |
|--------|----------|---------|-------------------|
| 论剑   | 每 2 周  | 2 次    | 每 2 周获得 160 张  |
| 诸天   | 每 2 周  | 2 次    | 每 2 周获得 160 张  |
| 宗门   | 每 1 周  | 1 次    | 每周获得 40 张      |
| 道蕴   | 每 3 周  | 1 次    | 每 3 周获得 40 张   |
| 百族   | 每 4 周  | 1 次    | 每 4 周获得 40 张   |

### 非战斗书页（狗粮池）

每个商店有一个狗粮池，汇总该商店所有非战斗神通的剩余书页。狗粮池有独立的收入周期，结构同战斗神通收入（周期 + 批次）。

狗粮池的用途：
- 作为金色材料直接消耗（无商店限制）
- 通过转换变成同商店战斗神通的本体书页

默认所有非战斗神通均为 1 星且无剩余书页，狗粮池初始为 0。

### 紫色 / 蓝色书页

紫色和蓝色书页为全局资源，每周固定收入（默认 0，可自定义）。

## 商店珍贵度（从高到低）

规划时优先消耗低珍贵度商店的书页，保留高珍贵度的。

```
百族 > 道蕴 > 诸天 = 宗门 > 论剑
```

## 功能需求

### 输入

#### 必填项

**6 个战斗神通**，每个输入：

| 字段         | 说明                             | 默认值   |
|-------------|--------------------------------|---------|
| 境界         | 从 6 个境界中选择                 | 人界一   |
| 职业         | 从该境界可用的职业中选择            | 剑      |
| 所属商店     | 从该职业可用的商店中选择            | 论剑     |
| 当前等级     | 1星 ~ 天5（受消耗分类限制）         | 1星     |
| 剩余本体书页 | 当前已有但未消耗的本体书页数量       | 0       |
| 目标等级     | 最低要达到的等级（受消耗分类限制）    | 天1     |
| 收入周期     | 每 N 周获得收入                   | 按商店默认 |
| 收入批次     | 每次获得 M × 40 张               | 按商店默认 |

> **境界 + 职业**决定消耗分类（影响升级消耗表和最高等级），**商店**决定转换来源和金色消耗优先级。

**通用材料**（2 个数字）：

| 字段     | 默认值 |
|---------|--------|
| 紫色书页 | 0      |
| 蓝色书页 | 0      |

#### 高级设置（均有默认值，可折叠隐藏）

| 字段                    | 默认值 | 说明                                                         |
|------------------------|--------|-------------------------------------------------------------|
| 转换石数量              | 0      | 可额外执行转换的道具                                           |
| 每周免费转换次数         | 3      | 跨所有商店共享的每周免费转换次数                                  |
| 狗粮池（每商店）         | 0      | 非战斗神通的剩余书页按商店汇总，含初始页数、收入周期和收入批次       |
| 紫色书页每周收入         | 0      | 每周额外获得的紫色书页数量                                      |
| 蓝色书页每周收入         | 0      | 每周额外获得的蓝色书页数量                                      |

### 输出

按周分步的操作规划，每周包含：

1. **收入结算**：本周各技能和狗粮池收到的书页，紫色/蓝色书页增量
2. **转换操作**：本周进行哪些转换（同商店狗粮池或技能盈余 → 目标战斗神通，含免费次数和转换石使用）
3. **升级操作**：本周可以进行哪些升级，具体消耗：
   - 本体书页消耗
   - 哪个来源（狗粮池/技能）的书页作为金色材料，消耗多少
   - 紫色/蓝色书页消耗
4. **状态快照**：6 个战斗神通的当前等级和剩余书页、各狗粮池余量、紫色/蓝色书页余量、剩余转换石

### 优化策略

1. **首要目标**：保证 6 个战斗神通全部达到用户设置的最低目标等级
2. **次要目标**：在达成最低目标后，利用剩余资源继续提升 6 个战斗神通的总体等级（穷举搜索所有 bonus 组合，取总等级提升最大的可行方案）
3. **迭代优化**：当逐周模拟因每周转换上限超出预期周数时，额外周数带来的收入可能支持更高的 bonus 等级。引擎会用实际周数重新运行 bonus 搜索并重新模拟，迭代直到收敛
4. **消耗原则**：优先消耗低珍贵度商店的书页池作为金色材料（论剑 > 诸天/宗门 > 道蕴 > 百族）
5. **转换原则**：优先从同商店狗粮池转换，其次从同商店技能盈余转换
6. **转换槽位优先级**：每周优先使用前 3 次免费转换 → 再使用转换石 → 最后使用剩余的免费转换次数（第 4 次及以后）
7. **升级无周次数限制**：材料充足即可连续升级，不受每周次数约束

### 规划终止条件

当 6 个战斗神通全部达到最终目标等级（包含 bonus）后，停止规划。

### 目标不可达处理

当用户设定的目标在可预见的资源范围内无法全部达成时：

- 明确提示哪些目标不可达及原因（缺本体、缺金色、缺紫色/蓝色、转换次数不足）
- 不展示规划方案

## UI 设计建议

### 输入界面

- **战斗神通卡片**：6 张卡片，每张含境界/职业/商店选择 + 等级选择器 + 书页输入 + 目标等级 + 收入设置
- **通用材料区**：紫色书页、蓝色书页各一个输入框
- **高级设置（可折叠）**：转换石、免费转换次数、狗粮池、每周收入等，全部带默认值

### 输出界面

- 时间线/步骤视图展示多周计划
- 每步操作可折叠展开，显示详细材料消耗
- 进度条展示目标达成情况
- 每周资源变化汇总（消耗了多少、剩余多少）
- 支持导出规划方案为 TXT 文件
