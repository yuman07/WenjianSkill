# 问剑长生 - 神通升级规划工具

## 项目概述

为手游《问剑长生》开发一个跨平台桌面应用（macOS / Windows），帮助玩家规划神通升级路径。用户输入当前所有神通的状态和目标，应用输出按周的最优操作步骤。

技术栈：**Tauri 2**

---

## 游戏数据模型

### 境界（6 个）

| 序号 | 境界名 |
|------|--------|
| 1    | 人界一 |
| 2    | 人界二 |
| 3    | 返虚   |
| 4    | 合体   |
| 5    | 大乘   |
| 6    | 渡劫   |

### 职业

- **通用职业**（所有 6 个境界均有）：剑、火、雷
- **百族职业**（仅返虚、合体、大乘 3 个境界有）

### 获取商店（4 + 1）

每个通用职业的 4 个神通分别来自不同商店：

| 商店   | 适用职业       |
|--------|---------------|
| 论剑   | 剑、火、雷     |
| 诸天   | 剑、火、雷     |
| 宗门   | 剑、火、雷     |
| 道蕴   | 剑、火、雷     |
| 百族   | 百族（专属）   |

### 神通总数

- 通用：6 境界 × 3 职业 × 4 神通 = **72**
- 百族：3 境界 × 1 职业 × 4 神通 = **12**
- **合计：84 个神通**

### 神通命名规则

`{境界}.{职业}.{商店}`，例如：`返虚.剑.论剑`、`合体.百族.百族`

## 神通等级体系

共 15 个等级，从低到高：

```
1星 → 2星 → 3星 → 玄1 → 玄2 → 玄3 → 地1 → 地2 → 地3 → 天1 → 天2 → 天3 → 天4 → 天5
```

### 升级消耗表

每次升级需要 4 种材料：

| 升级路径     | 本体书页 | 仙品书页 | 紫色书页 | 蓝色书页 |
|-------------|---------|---------|---------|---------|
| 获取（1星）  | 40      | 0       | 0       | 0       |
| 1星 → 2星   | 40      | 0       | 100     | 200     |
| 2星 → 3星   | 0       | 120     | 100     | 200     |
| 3星 → 玄1   | 40      | 120     | 150     | 350     |
| 玄1 → 玄2   | 80      | 120     | 200     | 500     |
| 玄2 → 玄3   | 80      | 160     | 250     | 650     |
| 玄3 → 地1   | 120     | 200     | 350     | 900     |
| 地1 → 地2   | 120     | 240     | 500     | 1200    |
| 地2 → 地3   | 160     | 280     | 600     | 1500    |
| 地3 → 天1   | 200     | 360     | 700     | 1800    |
| 天1 → 天2   | 240     | 440     | 800     | 2100    |
| 天2 → 天3   | 280     | 520     | 1000    | 2400    |
| 天3 → 天4   | 320     | 600     | 1000    | 2400    |
| 天4 → 天5   | 360     | 680     | 1000    | 2400    |

### 材料说明

| 材料     | 说明                                                   |
|---------|-------------------------------------------------------|
| 本体书页 | 该神通自身的专属书页                                      |
| 仙品书页 | **任意**其他神通的本体书页均可充当，无职业/境界/商店限制        |
| 紫色书页 | 通用材料，全局共享                                        |
| 蓝色书页 | 通用材料，全局共享                                        |

---

## 转换机制

- **作用**：将一个神通的本体书页转换为同商店另一个神通的本体书页
- **限制**：仅限**同商店**内转换（如 `返虚.剑.论剑` ↔ `大乘.雷.论剑`，但不能转为 `大乘.雷.诸天`）
- **比例**：1:1 无损（投入 40 张，获得 40 张）
- **数量**：每次固定转换 **40 张**
- **频率**：每周最多 **6 次**（跨所有商店共享次数）
- **转换石**：消耗 1 个转换石可额外执行 1 次转换，不受每周 6 次限制，使用次数无上限

---

## 每周商店收入

每个商店每周可获取一次神通（即 +40 本体书页），获取次数默认为 **1 次/商店/周**，用户可自定义。

规划时需考虑：每周新增的书页应获取到哪个神通上（由规划算法推荐）。

---

## 商店珍贵度（从高到低）

规划时优先消耗低珍贵度商店的书页，保留高珍贵度的。

```
百族 > 道蕴 > 诸天 = 宗门 > 论剑
```

## 功能需求

### 输入

#### 1. 战斗神通（6 个，逐个输入）

从 84 个神通中选择 6 个作为战斗神通，每个需输入：
- 具体是哪个神通（境界.职业.商店）
- 当前等级（1星 ~ 天5）
- 当前剩余本体书页数量
- 最低目标等级

#### 2. 非战斗神通书页（按商店汇总，5 个数字）

选定 6 个战斗神通后，其余非战斗神通的书页按所属商店汇总输入：

| 商店 | 输入内容                                      |
|------|----------------------------------------------|
| 论剑 | 该商店下所有非战斗神通的本体书页总和              |
| 诸天 | 同上                                          |
| 宗门 | 同上                                          |
| 道蕴 | 同上                                          |
| 百族 | 同上                                          |

> **简化原理**：非战斗神通的书页只有两个用途——作为仙品材料直接消耗（无商店限制）或通过转换变成战斗神通的本体书页（限同商店）。同商店内的非战斗书页完全等价，因此只需汇总数量。
>
> **数量约束**：所有书页数量（包括商店池和战斗神通的本体书页）始终为 **40 的整数倍**。

#### 3. 通用材料

- 紫色书页总量
- 蓝色书页总量
- 转换石数量

#### 4. 每周收入（可选，均有默认值）

| 收入项             | 默认值 | 说明                              |
|-------------------|--------|----------------------------------|
| 论剑商店获取次数    | 1      | 每次 +40 本体书页                  |
| 诸天商店获取次数    | 1      | 同上                              |
| 宗门商店获取次数    | 1      | 同上                              |
| 道蕴商店获取次数    | 1      | 同上                              |
| 百族商店获取次数    | 1      | 同上                              |
| 紫色书页每周收入    | 0      | 每周额外获得的紫色书页数量           |
| 蓝色书页每周收入    | 0      | 每周额外获得的蓝色书页数量           |

### 输出

按周分步的操作规划，每周包含：

1. **商店获取**：本周每个商店的书页应获取到哪个神通上
2. **转换操作**：本周进行哪些转换（源商店池 → 目标战斗神通，含免费次数和转换石使用）
3. **升级操作**：本周可以进行哪些升级，具体消耗：
   - 哪个商店池的书页作为仙品材料，消耗多少
   - 紫色/蓝色书页消耗多少
4. **进度概览**：6 个战斗神通的当前等级 vs 目标等级

### 优化策略

1. **首要目标**：保证 6 个战斗神通全部达到用户设置的最低目标等级
2. **次要目标**：在达成最低目标后，利用剩余资源继续提升 6 个战斗神通的总体等级
3. **消耗原则**：优先消耗低珍贵度商店的书页池作为仙品材料（论剑 > 诸天/宗门 > 道蕴 > 百族）
4. **转换原则**：优先转换低珍贵度商店的书页来满足战斗神通的本体书页需求
5. **转换石使用**：当每周 6 次免费转换不足以达成最优方案时，合理分配转换石
6. **升级无周次数限制**：材料充足即可连续升级，不受每周次数约束

### 目标不可达处理

当用户设定的目标在可预见的资源范围内无法全部达成时：
- 明确提示哪些目标不可达及原因（缺本体、缺仙品、缺紫色/蓝色）
- 展示在当前资源下能达到的最佳方案

## UI 设计建议

### 输入界面

- **战斗神通选择器**：下拉/搜索从 84 个神通中选取 6 个，每个配等级选择器 + 书页数输入 + 目标等级选择器
- **非战斗书页池**：5 个商店各一个数字输入框（自动排除已选为战斗神通的商店中对应书页的提示）
- **通用材料区**：紫色书页、蓝色书页、转换石各一个输入框
- **每周收入设置**：5 个商店各一个数字输入框，带默认值 1

### 输出界面

- 时间线/步骤视图展示多周计划
- 每步操作可折叠展开，显示详细材料消耗
- 进度条展示目标达成情况
- 每周资源变化汇总（消耗了多少、剩余多少）
